OD Toolbox R2013b (v6.5) Release Notes

Nov 27, 2013

 

The following sections describe the New Features, the Changes to Existing Features, the Known Bugs, and Notes in OD Toolbox Release 2013b (v6.5) since the previous release.

 

 

 

NEW FEATURES:

 

Estimator Functions:

 

 ·         Added a new object-oriented Solve-For/Consider parameter framework. Parameters include solar radiation, gravity models for Earth/Sun/Moon, measurement biases, and atmosphere (troposphere/ionosphere) distortion. Additional parameters will be added in future ODTBX releases.

 

 

Measurement Functions and Link Budget Modeling:

  

·         The Link Budget modeling capability has been abstracted out of gpsmeas into its own set of functions (linkbudget, get_linkbudget, etc.). These new functions are now called from gpsmeas, gsmeas, and tdrssmeas_basic. This provides a common link budget functionality to GPS, Ground Station, and TDRSS measurement functions.

 ·         TDRSS measurement modeling has been redesigned. The new tdrssmeas_basic function contains geometric modeling and the option for basic ionosphere modeling. This will eventually replace tdrssmeas in a future ODTBX release.

 ·         Added light time-of-flight calculations to gpsmeas via an odtbxOptions setting. See gpsmeas_example for usage.                                                                                                                     

 ·         Added ability to read Rinex files via the read_rnxn function. Using light time-of-flight requires a Rinex input file.                                                                                                                           

 

 

Force Modeling Functions:

 

·         (no significant changes in this release)

 

 

Other Functions:

 

·         The new AutoDX object implements an algorithm to automatically compute the finite-difference gradient with the lowest error for a given function. It can also be used to determine the amount of error in the function’s output as well as the optimal finite-difference step size. In a future ODTBX release, This will replace numjac as the primary tool to compute Jacobian matrices.

·         The new Apps folder contains three MATLAB apps that can be installed if MATLAB R2012b or later is used. The “Relative Motion Tool” app aids in the visualization of relative motion between two spacecraft. The “AllanHadamard Variance Tool” allows visual evaluation of tracking data with various variance measures (including Allen, Hadamard, and FFT). The “Measurement Scheduling Tool” allows ground station measurements to be scheduled with an easy-to-use graphical interface.

 

 

 

CHANGES OR FIXES TO EXISTING FEATURES:

 

Estimator Functions:

 

 ·        (no significant changes in this release)

 

  

Measurement Functions and Link Budget Modeling:

 

·         Fixed bugs in tdrssmeas_basic related to Doppler signal mixing, 1/2-way measurement differences, and more.

·         The basic troposphere model now checks if the transmitter/receiver are on the ground.

 

 

Force Modeling Functions:

 

·         (no significant changes in this release)

 

 

Other Functions:

 

·         Fixed incorrect calculation of pixel mapping matrix in CCDCamera.

·         Fixed bug related to looking up path variables in default startup.m.

·         The evtfun argument to integev will be deprecated in a future version of ODTBX. Users should instead specify an event function through the OdtbxOptions structure and call integ for all numerical integration.

 

 

 

KNOWN BUGS:

 

·         The chaser_target_tutorial function has a bug related to incorrect usage of the new link budget function. A patch will be released on the ODTBX download website when available.

·         The montecarloseed function may have slight variations in results when run on different architectures (specifically, on different Intel processors). The variations are very small and do not significantly affect results.

·        Problems with a dual-monitor setup in MATLAB with Windows:

An existing MATLAB bug may affect the way estval and some other ODTBX plots routines display the plots when using dual monitors under Windows.  The MATLAB bug is #301358 (Mathworks account required) and occurs in all recent MATLAB versions, R14 (7.0) through 2012b (8.0) and may affect different graphics drivers differently.

·         gpsmeas does not properly apply the acquisition and tracking thresholds when different values are used with a sequential estimator (a warning is produced). These acquisition/tracking thresholds were written with the batch estimator in mind and do not currently have consistent behavior when used with sequential estimators.  The acquisition and tracking thresholds are now defaulted to the same value to ensure consistent default operation on either estimator.

 

 

 

NOTES:

 

·         Other MICE versions: If required, a different version of MICE can be used with MATLAB and ODTBX than the ones supplied with this ODTBX installer.  Frequently this is required when a particular platform is not supported by the ODTBX installer.  MICE and SPICE can be downloaded from http://naif.jpl.nasa.gov/naif/toolkit.html.  Install MICE into a directory under the ODTBX installation directly and name it “mice”.  The startup.m supplied with ODTBX will look for MICE paths in that directory.  ODTBX paths also expect a “mice/kernels” subdirectory as a convenient place for SPICE kernels to live.  Create that subdirectory as well, even if you don’t intend to use it, so that all of the ODTBX MICE paths will be found.  You will have to download your own SPICE kernels but the pck00009 (or later), the naif0009 (or later), and the de421 kernels are recommended.
