OD Toolbox 5.0 Release Notes

August 31, 2012

 

The following sections describe the New Features, the Changes to Existing Features, the Known Bugs, and Notes in OD Toolbox Release 5.0 since the prior Release 4.5.

 

 

 

NEW FEATURES:

 

Estimator Functions:

 

·         Added initialize_cov: computes an initial covariance matrix based on energy and angular momentum.

 

·         Added estpar: A Bayesian particle filter estimator that calculates the state estimate based on "point-mass" or particle representations of probability.

 

·         Added sensmos: plots a "sensitivity mosaic," which is a checkerboard plot of the sensitivity matrix. This was originally a part of lincov_kf, but is now standalone.

 

 

Measurement Functions:

 

·         Added opt_sched: provides the ability to iteratively (and visually) optimize a measurement schedule.

 

·         Several clock error measurement models have been added, for use with one-way measurement types such as GPS.

·         clkdyn2: a 2-state clock error dynamics model.

·         clkdyn3: a 3-state clock error dynamics model.

·         clkprop2: a 2-state clock error propagator.

·         clkprop3: a 3-state clock error propagator.

 

·         Added gpspseudomeas: combines a clock model and a GPS pseudomeasurement model (from gpsmeas). This allows the modeling of one-way measurements from GPS that are corrupted by user clock errors.

 

·         Added CCDCamera: a subclass of Camera that converts landmark positions in the camera focal plane to pixel/line locations in the CCD. This allows the modeling of pixel mapping and distortion effects within the current OpNav measurement model (in opnavmeas).

 

 

GPS Measurement Functions and Antenna Link Budget Modeling:

  

·         (no significant changes in this release)

 

 

Force Modeling Functions:

 

·         Added nbodypm: an n-body force model that implements point-mass dynamics equations.

 

·         Added polydyn: a polyhedral gravity force model. This allows for more accurate gravity modeling for asteroids.

 

·         Added gasdyn: a comet-outgassing model that uses a spherical harmonics model and the position of the sun to determine the pressure gradient about the comet.

 

 

Other Functions:

 

·         Added plotAttitude() function to Attitude class. This produces a 3D plot of the basis vectors for an attitude rotation matrix.

 

·         Three new demos have been added to the ODTBX Example Listing section of the MATLAB Help pages:

·         pancake_demo: This demo shows how to compute observability and perform batch least squares estimation with ODTBX.

·         batch_3gs_demo: This demo shows how to compute observability using the ODTBX function estbat, and includes 3 ground stations observing a satellite.

·         chaser_target_demo: This demo shows the estimation of relative position in a chaser-target situation, using measurements and gps on the chaser.

 

 

 

CHANGES OR FIXES TO EXISTING FEATURES:

 

Estimator Functions:

 

 ·        Updated estbat to use a QR decomposition instead of basic matrix inversion for monte carlo solutions. The new method (similar to that used in lincov) produces a much more reliable solution for long-arc trajectories with weak observability.

 

 ·        estbat no longer assumes a linear approximation for the state propagation in the Monte Carlo error analysis.

 

 ·        The updateIterations input for estbat now specifies a maximum number of iterations (instead of a fixed number). estbat will now iterate only as many times as necessary to obtain a certain degree of accuracy.

 

 

Measurement Functions:

 

·         (no significant changes in this release)

 

  

GPS Measurement and Antenna Link Budget Modeling:

 

·         (no significant changes in this release)

  

 

Force Modeling Functions:

 

·         (no significant changes to force models for this release)

 

 

Other Functions:

 

·         The integ function now handles events. This allows for stopping integration at boundaries, peri/apogee, or any other user-defined event.

 

·         Overhauled the internal regression testing mechanism. Regression tests are now required to follow a specific naming convention, and can no longer be contained within the main ODTBX function itself. This allows regression tests to be removed from releases and keeps the codebase clean.

 

·         The estbat function now correctly handles infinite initial covariances.

 

·         The gmatforces_km function now accepts the ODTBX options structure as an optional input.

 

·         The July 1 leap second was added to the JAT and MICE leap-second tables, and all associated source files and regression tests have been updated to reflect the change.

 

 

 

KNOWN BUGS:

 

·         The montecarloseed function may have slight variations in results when run on different architectures (specifically, on different Intel processors). The variations are very small and do not significantly affect results.

 

·        Problems with a dual-monitor setup in MATLAB with Windows:

An existing MATLAB bug may affect the way estval and some other ODTBX plots routines display the plots when using dual monitors under Windows.  The MATLAB bug is #301358 (Mathworks account required) and occurs in all recent MATLAB versions, R14 (7.0) through 2011a (7.12) and may affect different graphics drivers differently.

 

·         gpsmeas does not properly apply the acquisition and tracking thresholds when different values are used with a sequential estimator, see above.

 

  

 

NOTES:

 

·         Other MICE versions: If required, a different version of MICE can be used with MATLAB and ODTBX than the ones supplied with this ODTBX installer.  Frequently this is required when a particular platform is not supported by the ODTBX installer.  MICE and SPICE can be downloaded from http://naif.jpl.nasa.gov/naif/toolkit.html.  Install MICE into a directory under the ODTBX installation directly and name it mice.  The startup.m supplied with ODTBX will look for MICE paths in that directory.  ODTBX paths also expect a mice/kernels subdirectory as a convenient place for SPICE kernels to live.  Create that subdirectory as well, even if you dont intend to use it, so that all of the ODTBX MICE paths will be found.  You will have to download your own SPICE kernels but the pck00009 (or later), the naif0009 (or later), and the de421 kernels are recommended.
